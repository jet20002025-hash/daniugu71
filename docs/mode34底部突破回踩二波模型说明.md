# mode34 模型说明（底部突破回踩二波 · 电科严格模版）

> **电科数字 600850 时间轴（交易日）**  
> **5/22 列入观察**（周五收盘，对应周末 5/24 维护观察清单）→ **5/25 信号日/最佳买点（预案）** → **5/26 盘中突破昨高执行买入**（二波确认仅作验证，不作信号日）

**无宽松版。** 扫描输出的 `signal_date` = **预案买点日**，不是二波确认日。

---

## 一、三日流程

| 阶段 | 电科日期 | 含义 |
|------|----------|------|
| **观察日** | **5/22** | 上一交易日入池；5/24 休市，清单在 24/25 前维护 |
| **信号日（买点）** | **5/25** | 收阳+缩量+不破铁底，收盘后给「偏多买入」 |
| **执行买入** | **5/26** | 盘中突破 5/25 最高价（触发价≈昨高） |

```text
5/22 观察 → 5/24(休市) 盯盘 → 5/25 预案/信号 → 5/26 盘中买
```

---

## 二、统一硬条件（启动段）

| 条件 | 阈值 |
|------|------|
| 底部位置 | ≤ 12% |
| 底→峰涨幅 | ≤ 35% |
| 12 月新低 | 启动段最低价为 12 月内最低 |
| 涨停启动 | 铁底→峰至少 1 日涨停，量比 ≥ 2 |
| 6 月最大量 | 铁底→峰含 6 月内最大成交量 |

**观察日额外：** 当日未出预案信号；**下一交易日**即为首个预案买点日（电科：22 的下一交易日是 25）。

**预案信号日（偏多买入）：** 必须 **收阳 + 缩量（量比≤0.75）** + 综合分≥72。

## 三、脚本

```bash
# 观察日
python3 scripts/scan_mode34_watch_prebuy.py --watch-date 2026-05-22

# 信号日（预案买点，非 5/26 确认日）
python3 scripts/scan_mode34_today.py --date 2026-05-25 --code 600850
python3 scripts/scan_mode34_period.py --start 2026-05-01 --end 2026-05-31

# 观察池整月
python3 scripts/scan_mode34_watch_period.py --start 2026-05-01 --end 2026-05-31
```

输出字段：`watch_date`、`signal_date`、`exec_buy_date`、`confirm_date`（若有二波阳）。

实现：`match_mode34_prebuy_signal`（信号）、`match_mode34_watchlist`（观察）。
