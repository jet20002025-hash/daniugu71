# 板块趋势与资金集中度（扫描增强）

在量价信号之外，增加**信号日当天及以前**的行业板块指数走势，以及可选的**当日资金流向排名快照**。

## 数据从哪来

| 内容 | 路径 | 说明 |
|------|------|------|
| 个股所属行业 | `data/gpt/akshare_cache/industry/{code}.txt` | 一行文字，东财「所属行业」 |
| 行业指数日 K | `data/gpt/akshare_cache/industry_hist/{行业}.csv` | `ak.stock_board_industry_hist_em` |
| 资金流向排名快照（可选） | `sector_flow_rank/{日期}.json` | `fetch_sector_flow_snapshot.py`；**当日整榜**排名 |
| **同花顺多窗口资金特征**（选股融合） | `sector_flow_ths_features.json` | `build_ths_sector_flow_features.py`；合并 **即时 / 5日 / 10日 / 20日** 行业榜 |

## 同花顺「最近一个月」说明

- 公开接口**不能按日历**拉「过去某一天」的整榜；**日历级历史**只能靠每天跑 `fetch_sector_flow_snapshot.py` 攒 `sector_flow_rank/YYYY-MM-DD.json`。
- **近似「近期资金偏好」**：用 **5日 / 10日 / 20日** 三个窗口的**当前**排行，与 **即时** 榜一起写入 `sector_flow_ths_features.json`，并计算 `flow_momentum = rank_20日 - rank_5日`（越大表示短线资金相对更集中该行业）。
- 扫描时**仅当** `signal_date` **等于** 特征文件里的 **`trade_date`** 才写入 metrics（避免区间回测把「今天的资金」用到历史信号上）。

## 如何准备缓存

```bash
# 1) 行业名 + 行业指数 K 线（耗时与股票数成正比）
python3 scripts/prefetch_sector_for_scan.py --sleep 0.15

# 2) 当日行业资金整榜快照（可选）
python3 scripts/fetch_sector_flow_snapshot.py

# 3) 多窗口特征（融入选股排序 / reasons；trade_date 须与扫描信号日一致）
python3 scripts/build_ths_sector_flow_features.py --trade-date 2026-03-23
# 依赖： pip install akshare py_mini_racer …
```

## 扫描里会多什么

- **metrics**：`industry`、`industry_ret5/10/20`（板块指数 N 日涨幅 %）、`sector_flow_rank`（按日快照文件）、`ths_flow_rank_1d/5d/10d/20d`、`ths_flow_momentum`、`ths_flow_net_*`（多窗口特征，**日期对齐时**）。
- **reasons**：行业名、板块指数、快照排行、同花顺 5 日/即时榜、资金相对走强等。
- **排序**（`mode3_sort_tuple`）：同分后依次考虑 **行业指数 5 日涨幅**、**同花顺 5 日资金榜名次**（越靠前越好）、**flow_momentum**（越大越好）。
- **总分里的板块热度**：不再按行业指数涨跌幅加分；**仅**通过 mode9 内 **信号日各行业涨停家数**（涨停行业 TopN、按家数加档等，`app/limit_up_industry_top.py`）体现，与「只统计涨停个数」一致。行业指数仍用于展示与排序 tie-break。

## 环境变量（Web 扫描）

- `SECTOR_AK_CACHE_DIR`：指定 akshare 缓存根目录；不设置则若存在 `data/gpt/akshare_cache/industry` 且其中有 `.txt` 则自动启用。

## 数据规则

- 行业 ret 仅用**不晚于信号日**的 K 线收盘，禁止用未来数据。
- 「资金流向排行」来自**磁盘快照**；非信号日当天文件则该项为空，避免误用「今天」资金代表历史某日。
